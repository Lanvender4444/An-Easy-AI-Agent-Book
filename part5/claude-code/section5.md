# 5. 两个 Agent 的对照与借鉴

拆完两个项目，把它们并排放在一起，能看到 Agent 工程的两种范式，以及一组可迁移的经验。

## 5.1 两种范式的对照

| 维度 | Open Notebook（垂直应用型） | Claude Code（通用 Harness 型） |
|------|---------------------------|------------------------------|
| **要解决的问题** | 一个具体场景：研究助手 | 通用：在任意代码库里干活 |
| **流程谁定** | 开发者（LangGraph 写死状态图） | 模型（开放式 Agent Loop 动态决策） |
| **自主程度** | 低，人在环中，流程确定 | 高，可后台自治、自我迭代 |
| **核心抽象** | 多 Provider（Esperanto）+ 领域模型 | Model + Harness（工具/权限/钩子/编排）|
| **工具** | 封装在图节点里（抽取/检索/TTS） | 开放工具集 + MCP 无限扩展 |
| **权限** | 弱（dev 级密码 + 凭据加密） | 强（allow/ask/deny + Hook 拦截 + 计划模式）|
| **记忆** | 用户的研究资料（领域数据） | 项目工程知识 + Agent 工作经验 |
| **编排** | 确定性工作流 + 异步队列 | 子智能体 / 后台 Agent / Agent 团队 |
| **扩展方式** | REST API + 自定义 Transformation | Slash 命令 / Skills / Plugins / MCP / SDK |
| **风险面** | 小（流程可控） | 大（自主行动，需重缰绳） |

一句话：**Open Notebook 是"把 AI 装进产品的一层"，Claude Code 是"把产品建在 Agent 之上"。** 前者的难点在工程组织，后者的难点在"给自主性套缰绳"。

## 5.2 为什么自主程度决定了架构重心

这是两章对比最值得记住的一点：**一个 Agent 的自主程度，决定了它的 Harness 把力气花在哪。**

- Open Notebook 自主程度低 → 流程开发者说了算 → 重心在**编排的确定性**（LangGraph 状态图 + checkpoint）和**抽象的复用性**（自研库下沉）。它几乎不需要权限层和感知层，因为模型不会"乱跑"。
- Claude Code 自主程度高 → 流程模型说了算 → 重心在**约束与安全**（权限三态、9 种 Hook、子智能体隔离、计划模式）和**可扩展性**（MCP/Skills/Plugins/SDK）。它必须假设"模型会做出意料外的动作"，于是把大量设计投在"如何安全地放权"。

> 这呼应第九章的核心论点：**Harness 的本质，是在"放权给模型"和"控制模型"之间找平衡。自主性越高，Harness 越重。**

## 5.3 可迁移的工程经验

把两个项目的优点合起来，给想做 Agent 的人一份清单：

1. **抽象掉 Provider**（学 Open Notebook 的 Esperanto / ModelManager）：统一接口 + 凭据治理 + 智能选择 + 失败兜底，别让 if/else 淹没业务。
2. **确定的归确定，自主的归自主**：稳定、可重复的环节用确定性工作流（LangGraph 那样）写死；需要灵活探索的环节才交给开放式 Agent Loop。两者可以混用。
3. **自主性必须配缰绳**（学 Claude Code）：只要让模型自己决定动作，就要上权限三态、动作前后钩子、危险操作拦截、计划模式。缰绳的强度要和自主度匹配。
4. **上下文是稀缺资源，要主动管理**：用 CLAUDE.md/auto memory 注入长期知识，用 compact/clear 管理短期窗口，用**子智能体把脏活隔离在独立上下文**里——这是 Claude Code 给的最大启发之一。
5. **把扩展点标准化**：MCP（工具）、Skills（隐式技能）、命令（显式工作流）、Hooks（生命周期）、Plugins（打包分享）——清晰的扩展分层让生态能长出来。
6. **长任务异步化、可恢复**：Open Notebook 用队列 + checkpoint，Claude Code 用后台 Agent + 会话持久化——都是为了让"跑得久的活"不阻塞、不丢失。
7. **一套核心、多个 surface**：把 Agent 的核心能力和入口解耦，配置（CLAUDE.md/settings/MCP）跨端复用。

> **收尾**：这两个项目一个偏"应用"、一个偏"基础设施"，恰好把本书前四部分的概念在真实代码里走了一遍——Open Notebook 让你看见 RAG、记忆、多 Provider 抽象如何工程化；Claude Code 让你看见 Agent = Model + Harness 如何在工业级产品里逐层落地。拆解真实项目，是把"知道"变成"会做"的最快路径。
