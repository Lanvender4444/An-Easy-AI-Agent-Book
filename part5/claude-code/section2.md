# 2. 引导层与记忆层

这两层放一起讲，因为在 Claude Code 里它们高度交织——都是"往模型上下文里注入什么"的问题。

## 2.1 引导层：System Prompt + CLAUDE.md + Output Styles

**System Prompt（系统提示词）**：引擎内置的基础行为设定，定义 Claude Code"是谁、怎么干活、用什么工具、遵守什么安全约束"。这部分由 Anthropic 维护，用户一般不直接改，但可以通过下面几样东西**叠加**自己的约束。

**`CLAUDE.md`：给 Agent 的说明书**。这是第九章"引导层"里 AGENTS.md 的 Claude Code 版本——一个放在项目里的 Markdown 文件，每次会话开始自动读入。你用它写：编码规范、架构决策、偏好的库、评审清单、命令怎么跑。它的精髓是**分层组合（第九章讲过的"全局 + 项目 + 子目录"）**：

- **用户级**（`~/.claude/CLAUDE.md`）：你个人跨所有项目的偏好。
- **项目级**（仓库根 `CLAUDE.md`）：团队共享、随仓库提交。
- **子目录级**（子目录里的 `CLAUDE.md`）：进入该目录作用域时叠加生效。

> 上一章我们就亲眼见过一个真实的 `CLAUDE.md`——Open Notebook 仓库根那份架构指南，正是写给 Claude Code 读的。它把三层架构、技术栈、"怎么加一条 LangGraph 工作流"都写清楚了，这样 Agent 一进项目就懂规矩。**这就是引导层的威力：用一个文件把"项目知识"固化下来，省掉每次重复解释。**

**Output Styles（输出风格）**：调整 Claude Code 的"人格/输出方式"。仓库 `plugins/` 里就有两个用 Hook 模拟已下线输出风格的例子——`explanatory-output-style`（讲解实现选择背后的考量）和 `learning-output-style`（在关键决策点让你亲手写 5–10 行代码）。这说明输出风格本质也是"往上下文注入引导信息"。

## 2.2 记忆层：CLAUDE.md（手写）+ Auto Memory（自动）

第十二章我们花了大力气讲 Agent 记忆。Claude Code 的记忆是**工程上下文记忆**，分两块：

**手写记忆 = CLAUDE.md**。它既是引导层也是长期记忆层——把"这个项目怎么 build、测试怎么跑、有哪些坑"沉淀进去，跨会话持久。对应第九章记忆层说的"CLAUDE.md / PROJECT_JOURNAL.md / 结构化交接"。

**自动记忆 = Auto Memory**。这是较新的能力：Claude 在干活过程中**自动**把学到的东西（build 命令、调试洞见等）存下来，跨会话复用，**你一个字都不用写**。这对应第十二章框架里的"信息提取 + 记忆管理"——只不过提取和管理由 Agent 自己完成，存的是"工程经验"而非"用户资料"。

> 把两章连起来看：Open Notebook 的记忆是**用户的研究资料**（领域数据记忆），Claude Code 的记忆是**项目的工程知识 + Agent 自己的工作经验**（任务/过程记忆）。两者都属"外部记忆"，但服务对象完全不同。

## 2.3 短期记忆与上下文管理：/compact 与 /clear

长会话会把上下文窗口塞满。Claude Code 的短期记忆管理对应第九章讲的两条路线：

- **压缩（/compact）**：把长对话历史**总结压缩**成精简版，保留关键信息、腾出窗口。这就是第九章"压缩"路线的实现。
- **重置 + 交接（/clear、resume）**：`/clear` 直接清空上下文重开；会话可被持久化、之后 `--resume` / `--teleport` 接力。配合 `CLAUDE.md` 这个长期记忆，做到"短期清空、长期不丢"——正是第九章"重置 + 交接"路线。
- **PreCompact Hook**：在压缩发生前还能插钩子（见下一节），让你控制压缩时机/内容。

引导层和记忆层合起来回答了一个问题：**模型每一轮"看到的上下文"是怎么拼出来的**——System Prompt 打底，CLAUDE.md（分层 + auto memory）注入项目知识，对话历史（经 compact 管理）提供近期上下文。接下来看模型拿到上下文后，能"动手"做什么。
