# 快速构建智能体 —— LangChain

官方的核心哲学是**收缩 API 表面积、强化运行时**。

聚焦于一个核心的 Agent Loop。



## 智能体

[LLM 智能体在循环中运行工具以实现目标](https://simonwillison.net/2025/Sep/18/agents/)

智能体会一直运行，直到满足停止条件。—— 遵循 ReAct 循环。

### 智能体的核心组成

+ 模型（静态/动态）
+ 工具
+ 系统提示此词
+ 调用 —— 调用其他智能体
+ 结构化输出
+ 短期记忆 —— 中间件 / 通过 `create_agent` 上的 `state_schema`

+ 流式传输
+ 中间件

### 智能体的高级用法

+ 守卫 —— 验证和过滤内容，基于 确定性/模型 的守卫
+ 运行时 —— LangChain 的 `create_agent` 实际上是在 LangGraph 的运行时环境下运行的。LangGraph 暴露了一个 `Runtime` 对象，包含上下文、存储和流式写入。通过工具和中间件管理。
+ 上下文工程
+ MCP
+ 人机交互
+ 多智能体
+ 检索
+ 长期记忆
