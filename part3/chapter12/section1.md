# Agent 记忆

**Agent Memory vs RAG**：Memory 聚焦"交互态"的会话内/跨会话动态信息；RAG 聚焦静态的外部知识库。

**Agent Memory vs LLM Memory**：LLM Memory 是模型预训练知识（静态、固化在权重里）；Agent Memory 是智能体运行时记忆（动态、随交互变化）。

基本工作流：**记忆存储（Storage）→ 记忆更新（Update）→ 记忆检索（Retrieval）**。

## 记忆流水线

**用户消息 → 信息提取 → 记忆管理 → 记忆存储 → 信息检索 → LLM 生成回复**

什么是记忆策略？

记忆策略 = 在"写入成本"和"读取成本"之间做分配，加工得越重，写入越贵、读取越省，但信息损失越大。

### 信息提取（Information Extraction）

+ 直接归档（零处理存原始消息+时间戳）
+ 摘要式提取（LLM 生成摘要或关键词/标签）
+ 图式提取（抽取 SPO 三元组建知识图谱）

#### 直接归档

本质是数据库里的**Append-only Log**，也就是数据库里的 WAL、事件溯源（event sourcing）那套思想。进来什么存什么，只附一个时间戳。

+ **加工程度**：零。把**全部解释成本推迟到读取时**——读的时候 LLM 得重新把原始文本读一遍、自己理解。

+ **保真度**：100%,**无损**。它不下任何赌注，所以永远不会"赌错"——未来无论问什么,原始信息都还在。

+ **写入成本**：近乎 0。
+ **读取成本：最高**。原始对话又长又噪，容易爆上下文窗口。可以搭配 RAG 技术压缩上下文。

+ **致命短板**：**不可扩展**。历史越长，这堆原始消息越没法直接喂进 LLM。而且原始闲聊里信噪比极低，检索质量差。



#### 摘要式提取

LLM 生成摘要 / 关键词标签。类似于有损压缩。



两个子形态：

- **自然语言摘要** → 擅长"叙事连续性/gist",适合维持对话主线、人物画像。
- **关键词/标签** → 擅长**结构化检索与路由**(按 tag 过滤、分类、走哪条路),它其实是给记忆建了一层稀疏索引。



#### 图式提取（抽 SPO 三元组建知识图谱）

**写入时做结构化抽取**——把文本变成 (主语 Subject, 谓语 Predicate, 宾语 Object) 三元组，拼成知识图谱。

就是上次讲的 Graph RAG 的索引阶，,原封不动搬到记忆系统。

**加工程度**:最重。除了 LLM 抽取,还要做**实体消歧/去重 + 图谱合并**(把"狼人""坏人"对齐到同一节点)。

**保真度**:另一种有损——**保留关系性事实,丢弃语气、叙事、情绪**。"(玩家A)-[投票给]->(玩家B)"留住了,但"他投得很犹豫"没了。

**写入成本:三者最高**。**读取成本:对关系型/多跳查询极低**,正好支持上次讲的多跳推理与全局聚合。

**独有的杀手锏——可变性与时间性**:这是图式相对前两者最被低估的优势。记忆和 RAG 文档最大的区别是,**记忆里的事实会随时间变化、互相矛盾**。"玩家A是好人"→后来翻盘"玩家A是狼人"。

- 直接归档:两条都存着,**矛盾被保留但没解决**,读取时得自己调和。
- 摘要式:得重新生成/覆盖,容易留下过期残留。
- **图式:能做精准的"边失效/更新"**,甚至给边打上时间戳(双时态建模,Graphiti/Zep 那类系统的核心),天然支持"截至第3天夜晚,A的身份状态是什么"这种时间性查询。

**短板**:抽取脆弱、schema 难设计,而且**不是所有东西都塞得进 SPO**(情绪、长篇叙事、模糊语义)。



### 记忆管理（Memory Management）

核心：活着的记忆。

记忆的总容量是有限的。

模拟人类记忆生命周期的五项核心操作：

1. 关联相关经验（语义/时间/上下文连接）本质：建立关系
2. 整合碎片化记忆（"日常记录→事件摘要→用户画像“） 本质：自底向上压缩
3. 跨层级转换（FIFO 与热度评分） 本质：分层流动
4. 更新现有记忆（规则驱动如遗忘曲线、LLM 驱动如 MemTree 聚合、Agent 驱动如 MemGPT 工具调用） 本质：就地演化
5. 过滤无效信息（基于使用频率或语义去重） 本质：主动遗忘



#### 关联相关经验（语义 / 时间 / 上下文连接）

孤立的记忆是死的。

关联就是给记忆之间架桥。



三种桥,对应三种"相关"：

- **语义关联**:意义相近就连。技术上就是 embedding 相似度——"玩家A可疑"和"玩家A发言矛盾"在向量空间靠近。这是**向量记忆**天然提供的。
- **时间关联**:时间相邻/有先后就连。"第2天投票"→"第3天出局",形成时间链。这是**直接归档(带时间戳)**天然提供的。
- **上下文关联**:同一情境/会话/事件里发生的就连。同一局游戏、同一轮对话里的记忆互相挂钩。



#### 整合碎片化记忆（"日常记录 → 事件摘要 → 用户画像"）

把低层、零散、具体的记忆，**自底向上地压缩、抽象成高层知识**。

日常记录→事件摘要→用户画像，或者说

原子事实→中层→高层画像。

本质是 **分层抽象(hierarchical abstraction)**， 类似 Graph RAG 的**社区摘要**，把细节聚合成一棵**摘要树**，越往上越抽象、越浓缩。 LLM 驱动的周期性"反思(reflection)"。



#### 跨层级转换（FIFO 与热度评分）

**FIFO(先进先出)**:纯按时间,最老的先被挤出热层。简单、无需算分,但**"老"不等于"没用"**——一条很久以前但极关键的记忆会被冤枉挤掉。这是 MemGPT 处理上下文窗口溢出的基础思路(像 OS 换页)。

**热度评分(hotness / recency-frequency-relevance)**:给每条记忆算一个动态分数,**高分留在热层,低分下沉到冷层**。分数通常综合三因素——



很符合直觉的一层，可以类比于缓存策略。

### 记忆存储（Memory Storage）

组织上分扁平式（FIFO/JSON）与分层式（MemoryOS 三层）；

表示上分向量存储（FAISS/Qdrant，主流）与图存储（MemTree 层次树、Zep 时序知识图谱）。



### 信息检索（Information Retrieval）

四种范式：

+ 词汇检索（BM25/Jaccard，精确匹配）
+ 向量检索（余弦相似度 + ANN，最主流，解决词汇失配）
+ 结构检索（Mem0ᵍ 子图构建、Zep 的 BFS 图遍历）
+ LLM 辅助检索（查询重构 + 实体识别，如 MemoChat）。
