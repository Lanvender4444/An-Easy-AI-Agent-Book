## 引导层

System Prompt、AGENTS.md、约束文件、ADR、目录作用域、分层组合 —— 限制 Agent 的行动范围

+ System Prompt —— 基础行为设定
+ AGENTS.md —— 给 Agent 的说明书
+ 约束文件 —— 明确边界
+ ADR —— 规则背后的意图
+ 目录作用域 —— 不同的文件作用域在哪
+ 分层组合 —— 全局规则 + 项目规则 + 子目录规则

## 感知层

测试套件、验证器、评估器、阶段门 —— 对引导层的事后核查

+ 测试套件 —— 机械性的二元判定
+ 验证器 —— 约束文件的执行端，规则约定
+ 评估器 —— LLM as judge
+ 阶段门 —— 每个阶段之间设一道"门",必须满足条件才能通过

## 生命周期层

Hooks、存档、恢复、超时管理、会话启动自动注入、上下文重置 —— 一次会话内部的流转状态

+ Hooks： 特定时机(工具调用前/后、会话开始/结束)插入自定义逻辑
+ 存档：存档。保留中间态
+ 恢复：恢复存档
+ 超时管理：防止无限循环
+ 会话启动自动注入：自动塞进上下文,无需 Agent 主动去读
+ 上下文重置：重置记忆

## 编排层

Supervisor、Pipeline、 Swarm —— 多个子智能体之间的编排

+ Supervisor  —— 一个中央 supervisor 智能体接收任务，将其分解为子任务，分配给各专家智能体，收集结果后综合输出最终答案。只有 supervisor 能看到全局。
+ Pipeline —— 各智能体串联成线性链路，每个节点接收上游输出，完成自身的转换处理后传递给下游。
+ Swarm —— 各智能体点对点通信，根据当前状态动态移交工作。核心机制是 handoff：某个智能体判断自身已不再适合处理当前状态，便将控制权连同对话上下文一起转交给另一个智能体。
